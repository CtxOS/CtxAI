## How to get started
1. Find an issue labeled `good first issue` or `area:*` that matches your expertise.
2. Comment on the issue to claim it.
3. Fork the repo, implement your changes with tests, and open a PR.

## Labels
We use labels to help contributors find work quickly:
- `good first issue` for onboarding tasks.
- `area:*` for subsystem classification.
- `type:bug`, `type:enhancement`, `status:needs triage`, `status:ready`.

## Discussions
We maintain discussions for:
- General questions
- Feature planning
- Community announcements

If you're a contributor, join our Discord or start a discussion in GitHub Discussions.
